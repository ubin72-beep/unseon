1

카카오페이 가맹점 신청

결제를 받으려면 카카오페이 사업자 가맹 심사를 먼저 통과해야 합니다.

사전 준비물: 사업자등록증, 통장 사본, 대표자 신분증 (스캔본). 심사 기간은 보통 3~7 영업일.
1
카카오페이 파트너스 접속

business.kakao.com/info/kakaopay/ → 상단 "가맹점 신청" 버튼 클릭

2
서비스 유형 선택

온라인 결제 → "단건 결제" 선택 (정기결제는 구독 서비스 전용)

3
사업자 정보 입력

상호: 큐브박스 · 사업자번호: 537-08-03349 · 서비스명: 운세ON · 서비스 URL: https://unseon.co.kr

4
심사 완료 후 CID 발급

이메일로 CID (예: CZ0ONETIME) 와 Admin Key 가 전달됩니다. 반드시 안전하게 보관하세요.

운세ON 사업자 정보 (537-08-03349)는 이미 코드에 입력되어 있어 신청 시 그대로 사용하면 됩니다.
2

코드 설정 수정 (3줄)

js/kakaopay.js 파일 상단 KAKAO_CONFIG 딱 3줄만 수정하면 됩니다.

Admin Key는 절대 프론트엔드 코드에 노출하면 안 됩니다. CID만 프론트엔드에 입력합니다.

📄 js/kakaopay.js 파일을 열고 아래와 같이 수정하세요:

js/kakaopay.js — KAKAO_CONFIG (파일 맨 위) 
const KAKAO_CONFIG = {
  /* ① 테스트 → 실서비스 전환 */
  IS_TEST: false,                    // ← true 에서 false 로 변경

  /* ② 가맹 심사 후 발급된 CID 입력 */
  CID: '여기에_발급받은_CID_입력',  // ← 예: 'CZ0ONETIME'

  /* ③ 실제 서비스 도메인 (이미 설정됨) */
  ORIGIN: 'https://unseon.co.kr',    // ← 이미 올바르게 설정됨 ✅
};
도메인은 이미 설정됨! ORIGIN은 https://unseon.co.kr으로 이미 입력되어 있어 변경 불필요합니다.

변경 전 / 후 비교

❌ 변경 전 (테스트 모드)
IS_TEST: true,
CID: 'TC0ONETIME',
✅ 변경 후 (실서비스)
IS_TEST: false,
CID: '발급받은CID',

실결제 방식 선택 및 코드 활성화

정적 사이트에서 실결제를 처리하는 두 가지 방법 중 하나를 선택하세요.

방법 A — JS SDK

★ 권장

카카오에서 공식 제공하는 JavaScript SDK를 사용. 백엔드 서버 없이도 실결제 가능.

  • 백엔드 서버 불필요
  • 카카오 공식 방식
  • 설정이 간단함
  • 카카오 앱 키 필요

방법 B — 백엔드 API

서버에서 Ready API를 호출 후 결제 URL을 받아 이동. 보안성이 더 높음.

  • Admin Key 완벽 보호
  • 서버 검증 가능
  • Node.js/Python 서버 필요
  • 구축 비용 발생

방법 A — JavaScript SDK 적용 (권장)

① pricing.html <head>의 아래 주석을 해제하세요:

pricing.html <head> — SDK 스크립트 주석 해제 
<!-- 주석 해제 ( 제거) -->
<script src="https://t1.kakaocdn.net/kakaojs/sdk/2.7.4/kakao.min.js"></script>

② js/kakaopay.js → runRealKakaoPay() 함수 내 주석 코드를 해제하고 앱 키를 입력하세요:

js/kakaopay.js — runRealKakaoPay() 내 SDK 코드 활성화 
function runRealKakaoPay(plan) {
  const orderId     = generateOrderId();
  const currentUser = getCurrentUserInfo();

  sessionStorage.setItem('kakao_pending_plan', JSON.stringify({
    planKey: selectedPlan, plan, orderId
  }));

  // ★ Kakao SDK 초기화 (한 번만 실행)
  Kakao.init('여기에_카카오_앱키_입력');  // ← 카카오 개발자 앱 키

  Kakao.Pay.createOrder({
    open_type:        'REDIRECT',
    partner_order_id: orderId,
    partner_user_id:  currentUser?.id || 'guest',
    quantity:         1,
    total_amount:     plan.amount,
    tax_free_amount:  0,
    name:             plan.name,
    success_url: KAKAO_CONFIG.APPROVAL_URL + '&order_id=' + orderId,
    fail_url:    KAKAO_CONFIG.FAIL_URL,
    cancel_url:  KAKAO_CONFIG.CANCEL_URL,
  });
}
카카오 앱 키는 developers.kakao.com/console/app → 앱 선택 → 앱 키에서 "JavaScript 키"를 사용하세요.
방법 B — 백엔드 서버 연동 (Node.js 예시) 보기

서버에서 카카오페이 Ready API를 호출하고, 결제 URL을 프론트엔드로 전달하는 방식입니다.

서버 코드 (Node.js / Express)

server.js (Node.js)
const express = require('express');
const axios   = require('axios');
const app     = express();
app.use(express.json());

app.post('/api/payment/ready', async (req, res) => {
  const { orderId, planName, amount, userId } = req.body;
  try {
    const response = await axios.post(
      'https://open-api.kakaopay.com/online/v1/payment/ready',
      {
        cid:               process.env.KAKAO_CID,   // 환경변수로 보호
        partner_order_id:  orderId,
        partner_user_id:   userId,
        item_name:         planName,
        quantity:          1,
        total_amount:      amount,
        tax_free_amount:   0,
        approval_url:      'https://unseon.co.kr/payment-complete.html?result=success',
        fail_url:          'https://unseon.co.kr/payment-complete.html?result=fail',
        cancel_url:        'https://unseon.co.kr/payment-complete.html?result=cancel',
      },
      { headers: { Authorization: `SECRET_KEY ${process.env.KAKAO_ADMIN_KEY}` } }
    );
    res.json(response.data);  // tid, next_redirect_pc_url 반환
  } catch (e) {
    res.status(500).json({ error: e.message });
  }
});

프론트엔드에서 서버 호출

js/kakaopay.js — runRealKakaoPay() 수정
fetch('/api/payment/ready', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({

.then(r => r.json())
.then(data => {
  sessionStorage.setItem('kakao_tid', data.tid);
  window.location.href = isMobile()
    ? data.next_redirect_mobile_url
    : data.next_redirect_pc_url;
